SOFIDR · AUTOMATIC DATA PREPARATION

See how SOFIDR cleans your data — and why.

Before a model can learn, the data has to be cleaned and prepared. There are many ways to do that, and the best one depends on the data itself. SOFIDR tries the sensible options, scores each one, and picks the best — automatically. Below is a real run on a sample dataset.

Most teams choose a data-prep method out of habit. Choose wrong, and the model is weaker for no real reason. SOFIDR takes the guesswork out and shows its reasoning.
01

The sample data

We're looking at 1,000 customers from a subscription business. The goal is to predict who will cancel their plan. It's a realistic, tricky case — the outcome groups are very uneven, and some records are missing a value.

Sample · first 8 customers

Total charges is blank where the value is missing. Only about a fifth of customers churned, so a naive model could score "well" by simply predicting that nobody cancels — which is exactly the trap good data prep helps avoid.

Step 1 — Scan the data

SOFIDR measures the data before changing anything, so its choice fits what's actually there.

Step 2 — Line up the strategies

SOFIDR keeps a set of ready-made prep strategies. Each suits a different kind of data — there's no single method that's best for everything.

Step 3 — Score each one

Every strategy is tested the same way on the same data and given one score from 0 to 1. Higher is better.

The score (called SEI) blends four things — accuracy, consistency, how much data was kept, and how simple the result is. The full breakdown is explained below.

How the score works

Each strategy gets a single score, the Strategic Efficiency Index (SEI), from 0 to 1. It combines four things so no strategy can win on one measure while quietly hurting the rest.

40%
Accuracy
Is the model's prediction right?
25%
Consistency
Does it hold up on data it hasn't seen before?
20%
Data kept
Did it avoid throwing away too many records?
15%
Simplicity
Is the prepared data still easy to work with?

A method that's accurate but unstable, or accurate but discards half the data, scores lower overall. One combined number keeps the trade-offs honest.
